# 下个时代的组件库

这部分讨论 Vue 3 生态下更现代的组件库设计思路：  
**交互行为与视觉样式解耦**，让组件更易定制、更适配业务设计系统。

## 核心思路

1. 使用 Headless / Primitive 组件承载交互行为
2. 通过设计令牌（Design Tokens）统一主题能力
3. 样式层尽量交给应用侧（Tailwind / CSS Variables / CSS Layers）

## 为什么它重要

- 避免传统“大而全”组件库在复杂业务中的样式冲突
- 减少“覆盖第三方样式”的维护成本
- 更容易形成可演进的企业内设计系统

## 推荐阅读路径

- 先读本章中的组件使用与构建章节
- 再对比 headless 组件方案在交互和样式层的职责边界
- 最后结合你自己的业务场景，决定是“全量引入”还是“按需自建”
