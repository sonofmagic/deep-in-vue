# Svelte 组件生态速览

Svelte 提供了一个非常重要的参考视角：

> 如果把更多工作继续前置到编译阶段，组件库和框架本身的边界会发生什么变化？

这对读 Vue 文档很有价值，因为 Vue 本身就很强调“编译器不是附属品”。

## 可借鉴点

- 更激进的编译期优化策略
- 组件样式隔离与输出体积之间的权衡
- 对“框架成本”与“开发体验”的另一种平衡方式

## 对 Vue 的启发

它帮助我们反过来思考：

- 哪些能力更适合在编译期解决
- 哪些能力必须留在运行时
- 组件库是否应该继续变得更“源码导向”
